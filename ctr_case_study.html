Overview

A live government platform for
antimicrobial transaction tracking.

The Computerized Transaction Record (CTR) is an electronic web platform developed for the Hong Kong Department of Health, tasked by the High Level Steering Committee on Antimicrobial Resistance (HLSC). CTR provides licensed traders with a record-keeping system to record, view, and monitor antimicrobial transaction records — supporting the rational use and effectiveness monitoring of antimicrobials across Hong Kong.

My Contribution
Joined the project mid-development as a UI/UX designer. Responsible for redesigning existing page layouts and building virtually the entire component library in Figma — enabling the development team to construct new features rapidly through reusable, nested components.
The Scale
The application serves licensed traders, hospitals, pharmacies, wholesalers, manufacturers, and government health personnel across Hong Kong. The platform is currently live and in active use by the Department of Health.

Design System

Establishing the visual foundation —
colour, typography, and tokens.

Before building components, I established the design system's visual language. The colour scheme spans a green-teal primary palette with a full grey scale, status colours for warnings, errors, and success states, and specialised sets for inline messages and notification centre elements. Typography is set in Roboto across 10 defined header and body levels.

Colour scheme and typography scale — established as the foundation of the CTR component library
Component Library

116+ components built from scratch
in Figma.

As a UI/UX designer on the project, I built virtually every UI component used across the CTR platform — from atomic elements like buttons and icons to complex composite components like multi-step transaction flows, data tables with pagination, and responsive navigation systems. Each component was built with variants and nested structures so the development team could assemble new pages and features rapidly.
